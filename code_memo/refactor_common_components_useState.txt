- useState 활용도 향상 (검색 상태, 선택 상태 등)

--------------------------------------------------------------------------------
1. 신규 파일 — src/components/common/
--------------------------------------------------------------------------------

▸ PageLayout.jsx
  - Header + Sidebar를 포함한 표준 페이지 레이아웃 래퍼
  - props: navItems, activeItem(기본값 'Home'), className, children
  - 사용 페이지: SignupPage, UserManagePage, ProductManagePage

▸ FormField.jsx
  - Label + 입력 영역(children) + 성공 메시지를 묶는 폼 필드 래퍼
  - props: label, id, successMessage(falsy면 숨김), children
  - children으로 Input 또는 커스텀 요소(버튼 포함 flex 등)를 자유롭게 넣을 수 있음
  - 사용 페이지: SignupPage, UserManagePage

▸ PlayerCard.jsx
  - 매물 관리 카드 뷰 아이템 (체크박스, 아바타, 닉네임/소개, 팀 배지, 더보기 버튼)
  - props: item { id, nickname, desc, teams }, selected, onToggle, onMoreClick
  - 기존 ProductManagePage 내부 함수에서 독립 컴포넌트로 분리

▸ PlayerRow.jsx
  - 매물 관리 리스트 뷰 아이템 (체크박스, 아바타, 닉네임/소개, 팀 배지, 더보기 버튼)
  - props: item, selected, onToggle, onMoreClick
  - 기존 ProductManagePage 내부 함수에서 독립 컴포넌트로 분리

▸ SocialButton.jsx
  - 소셜 로그인 버튼 (Google / Kakao / Naver / Discord)
  - props: label(aria-label), className, svg, onClick
  - 기존 LandingPage 내부 SocialBtn 함수에서 독립 컴포넌트로 분리

  - 기존 useState 구조 유지 (form)

3. useState 사용 현황 정리
--------------------------------------------------------------------------------

  페이지                | state 변수          | 용도
  --------------------|--------------------|---------------------------------
  SignupPage          | form               | 입력 폼 전체 필드 값
                      | idChecked          | ID 중복 확인 완료 여부
  UserManagePage      | form               | 입력 폼 전체 필드 값
  NoticePage          | search             | 공지 검색어 (NEW)
                      | selectedId         | 선택된 공지 항목 ID (NEW)
  ProductManagePage   | view               | 카드/리스트 뷰 토글
                      | activeGame         | 선택된 게임 탭
                      | selected (Set)     | 선택된 플레이어 ID 집합
                      | search             | 플레이어 검색어 (NEW)
